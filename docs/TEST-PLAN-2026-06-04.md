# drag-lint graph viewer - follow-along test plan (2026-06-04)

A step-by-step pass you can run start to finish. Each step has an **action** and
what you **should see**. Tick the box if it matches; jot a note next to anything
that doesn't and send me the list.

Time: ~20-30 min for Parts A-E (viewer + IDE). Parts F-G (CLI + call graph) add
~10 min.

---

## 0. Setup (once)

- [ ] **0.1** RAD Studio is open with a project loaded, and the **drag-lint
      plugin BPL is installed** (Component -> Install Packages shows
      `dclDragLintWizard`, checked). This is what receives the "open source"
      jumps.
- [ ] **0.2** Launch the viewer from a terminal (the command you already used):

      C:\Projects\Delphi-RAG-Lint-Graph\bin\Win32\drag_lint_graph.exe ^
        --db C:\Projects\DB\ORM3\drag-lint.sqlite ^
        --db C:\Projects\Delphi-RAG-lint\third_party\dll-win32\drag-lint-library.sqlite

- [ ] **0.3** The window opens within a couple of seconds and is responsive
      (no hang), showing a handful of **collapsed unit boxes**, not a hairball.
- [ ] **0.4** Bottom-left shows a **legend** (the kinds present, colour-coded).
      Top-left shows a **breadcrumb** (`Project`). Top-right shows
      **Show all units (N hidden)** and a **Fit** button + zoom slider.

## 1. Navigation & layout

- [ ] **1.1 Pan with LEFT drag.** Press-drag on empty canvas -> the whole view
      pans and follows the mouse. Release -> pan stops, cursor returns to normal.
- [ ] **1.2 Pan with RIGHT drag.** Same thing with the right button. Works too.
- [ ] **1.3 No stuck "move mode".** After any pan, just move the mouse with **no
      button held** -> nothing pans, cursor is a normal pointer (not the 4-arrow
      move cursor).
- [ ] **1.4 Fit.** Click **Fit** (top-right) -> the whole graph is reframed to
      fit the window. Use this any time you get lost.
- [ ] **1.5 Zoom.** Mouse wheel over empty canvas zooms in/out around the
      pointer; the zoom slider tracks it. (Wheel over a box scrolls the box -
      see Part 3.)
- [ ] **1.6 Drill into a unit.** Single-LEFT-click a unit box -> the view zooms
      into just that unit's contents, and the breadcrumb becomes
      `Project > MSCTYPES` (or whichever unit).
- [ ] **1.7 Breadcrumb back.** Click the `Project` crumb -> you jump back out to
      the top level.
- [ ] **1.8 Mouse BACK button.** Drill into a unit again, then press the **mouse
      thumb "back" button** -> you go up one level. (Backspace key does the same.)
- [ ] **1.9 Even layout, no far-flung strays.** At the top level, units with no
      cross-unit links sit in a **tidy grid** beside the connected cluster - not
      scattered to the far corners.

## 2. Nodes, labels, and detail (the "what is this?" check)

Drill into **MSCTYPES** for this part (it is records + enums, good coverage).

- [ ] **2.1 Collapsed nodes are named.** Circles show a real name + member count,
      e.g. `TPlanType (+22)`, `AcReButton (+68)` - never a bare dot.
- [ ] **2.2 No "mystery hubs".** There are **no** points where many lines
      converge on an empty/unlabelled spot. Every line ends on a visible,
      named node.
- [ ] **2.3 Hover detail.** Rest the mouse on a node ~0.5s (don't move) -> a
      tooltip pops with the **precise kind** and name, e.g.
      `enum: MSCTYPES.TPlanType`, plus its signature/section/doc if any.
- [ ] **2.4 Status bar detail.** Click a node -> the bottom status bar shows
      `enum: MSCTYPES.TPlanType  (MSCTYPES.PAS:412:3)` (kind, qualified name,
      file:line:col). Enums/sets/arrays read as their real kind, not "Other".
- [ ] **2.5 Records as UML boxes.** A record (e.g. `SamplesRec`,
      `PartFileVariable`) shows as a titled **box** listing its fields with
      types (`+ Name: string`), `<<record>>` in the title.
- [ ] **2.6 Enums/types as boxes too.** Click an enum circle (or zoom in) -> it
      expands into a box listing its members, title `<<enum>> TPlanType`.
- [ ] **2.7 impl-only marker.** If any type is declared in a unit's
      implementation section, its box title shows `[impl-only]` and the hover
      says "not usable from another unit". (May not occur in MSCTYPES - fine.)
- [ ] **2.8 Visibility glyphs.** Member rows start with `+ public`, `- private`,
      `# protected`, `~ published`.

## 3. In-box scrollbar & resize

Pick a tall box (a record/enum with more members than fit, showing a scrollbar
on its right edge).

- [ ] **3.1 Wheel scroll.** Wheel over the box -> the member list scrolls; the
      view does **not** flicker (no full-screen redraw flash).
- [ ] **3.2 Scrollbar thumb.** Drag the **thumb** on the box's right edge -> the
      list scrolls to match. Click the track above/below the thumb -> it pages.
- [ ] **3.3 Resize grip.** Drag the **bottom-right grip** (small diagonal lines)
      down -> the box grows to show more rows; drag up -> it shrinks and the
      scrollbar appears. The size sticks for that box.
- [ ] **3.4 No flicker anywhere.** Pan / zoom / scroll - none of it flickers.

## 4. Selection, links, context menu

- [ ] **4.1 Select.** Click a node -> it gets a highlighted border; status bar
      updates.
- [ ] **4.2 Walk a link.** With a node selected, click a **link line** attached
      to it -> the node at the other end becomes selected. Click its next link
      -> hop again.
- [ ] **4.3 Deselect.** Click empty canvas (no drag) -> selection clears.
- [ ] **4.4 Right-click a node** -> menu: Open Source / Go to Interface /
      Where Used / Center Here / Back / Fit to Window.
- [ ] **4.5 Right-click empty canvas** -> menu still appears with **Back** and
      **Fit to Window** (never a dead click).
- [ ] **4.6 Center Here.** Right-click a node -> Center Here -> the view recenters
      on it.

## 5. IDE integration - open source (the big one)

IDE must be open with the plugin loaded (step 0.1).

- [ ] **5.1 Jump to source.** Click a **record** (or a member row inside a box)
      -> the IDE comes forward and opens that .pas at the **exact line** of the
      declaration. Status bar reads `Opened in IDE: ...PAS:line:col`.
- [ ] **5.2 Right place.** The caret lands on the actual symbol, not the top of
      the file and not a different file.
- [ ] **5.3 Member precision.** Open a box, click a single **member row** -> the
      IDE jumps to that member's line specifically.
- [ ] **5.4 Status feedback.** Every open shows `Opened in IDE: file:line:col`.
      If you ever see `Opened: ... (no IDE plugin listening)`, the pipe didn't
      reach the plugin - note it.
- [ ] **5.5 Context-menu open.** Right-click a node -> **Open Source** -> same
      jump.

> If 5.1 opens the file but lands on the **wrong line/file**, that is the bug the
> latest plugin guard fixes - tell me and confirm whether you rebuilt+reinstalled
> the plugin BPL.

## 6. CLI (optional, no IDE needed)

Run from a terminal. exe: `C:\Projects\Delphi-RAG-lint\third_party\dll\drag-lint.exe`

- [ ] **6.1 Find a symbol.**
      `drag-lint query --name TPlanType --db C:\Projects\DB\ORM3\drag-lint.sqlite`
      -> lists matches with kind + qualified name. Add `--json` to see
      `signature`, `section`, `usable_from_other_units`.
- [ ] **6.2 Which unit to `uses`?**
      `drag-lint resolve-uses --name <SomeType> --db C:\Projects\DB\ORM3\drag-lint.sqlite`
      -> ranked defining unit(s) + "Suggestion: add X to uses". Implementation-
      only symbols are flagged `<impl-only: NOT usable via uses>` and get no
      suggestion.
- [ ] **6.3 Callers.**
      `drag-lint query find-callers --name <Method> --db C:\Projects\DB\ORM3\drag-lint.sqlite`
      -> the call sites.
- [ ] **6.4 Syntax check (no compiler).**
      `drag-lint check-ast <some.pas>` -> reports `(line,col): error syntax-error`
      per bad spot (clean file -> nothing).

## 7. Methods & call graph

The data is there (the ORM3 index has ~19k methods and ~58k call edges).
MSCTYPES has none (pure data types), so use a class-heavy unit.

- [ ] **7.1** Go back to the top level (Fit / Back), then drill into a code unit
      such as **Blueprint4**.
- [ ] **7.2** Find a class (e.g. `TfrmBlueprint4`), click it -> it expands as a
      box listing its methods/fields (`+ Create`, etc.).
- [ ] **7.3 Call arrows.** With methods visible, look for **arrows between
      methods** - those are calls (method A -> method B). Walk them with 4.2.
- [ ] **7.4** Right-click a method -> **Where Used** -> the view dims to its
      1-hop neighbourhood (who it connects to).
